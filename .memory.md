## 出牌交互机制详解

### 信号流
`NHandCardHolder` 发出两个信号，都连接到 `NPlayerHand.OnHolderPressed` → `StartCardPlay(holder, startedViaShortcut: false)`：
- `HolderMouseClicked`：鼠标**按下**时发出（`NHandCardHolder.OnMousePressed` override）
- `NCardHolder.Pressed`：鼠标**松开**时发出（`NCardHolder.OnMouseReleased`，先 EmitSignal 再置 `_currentPressedAction = null`）

两个信号传入参数完全相同，在 `StartCardPlay` 层无法直接区分来源。

### NMouseCardPlay 内部短按/长按区分
`NMouseCardPlay.Start()` 中：`_isLeftMouseDown = !_skipStartCardDrag`（鼠标触发时初始为 `true`）

`_Input()` 收到左键释放事件后：`_isLeftMouseDown = false`

`StartAsync()` 流程：
1. `await StartCardDrag()`：逐帧跟随鼠标，循环直到鼠标 Y 坐标进入出牌区（屏幕上 75%）
2. 进入出牌区后检查 `_isLeftMouseDown`：
    - `true`（鼠标仍按着）→ `ReleaseMouseToTarget`（松手选目标，**长按拖拽**模式）
    - `false`（已松开）→ `ClickMouseToTarget`（点击选目标，**短按**模式）

### 区分短按/长按的可靠方法
在 `StartCardPlay` 入口处，用 `Godot.Input.IsMouseButtonPressed(MouseButton.Left)` 检测：
- 返回 `true`：鼠标仍按着 → 来自 `HolderMouseClicked`（按下）→ 长按拖拽
- 返回 `false`：鼠标已松开 → 来自 `Pressed`（松开）→ 短按结束

这与原游戏 `_isLeftMouseDown` 语义完全等价。
原游戏没有在 `StartCardPlay` 层区分的原因：两种触发方式传入参数相同，区分逻辑被推迟到 `NMouseCardPlay` 内部（进入出牌区时）。
`NCardHolder._currentPressedAction` 无法用于区分：`Pressed` 信号是在 `_currentPressedAction = null` 之前同步发出的，此时该字段仍非 null。

### ⚠️ 重要陷阱：两次 StartCardPlay 问题
每次点击实际上会触发**两次** `OnHolderPressed` → `StartCardPlay`：
1. 鼠标**按下** → `HolderMouseClicked` → `InCardPlay=false`，`CanPlayCards()=true` → 正常进入 `StartCardPlay`
2. 鼠标**松开** → `Pressed` → 若第1次已启动拖拽（`InCardPlay=true`），`CanPlayCards()=false`，**连 `StartCardPlay` 都不会到达**；若第1次被跳过（`InCardPlay=false`），则会再次进入 `StartCardPlay`

因此若在第1次出牌并跳过原方法，需要用标志位让第2次调用也跳过，防止重复出牌。

`NHandCardHolder.BeginDrag()`：将卡牌角度和缩放设为悬浮状态（视觉上"被拿起"）。
`NHandCardHolder.CancelDrag()`：恢复卡牌视觉（ZIndex=0，角度和缩放归零）。

## ClickToPlay Mod 实现

**Patch 文件**：`ClickToPlay/Patches/ClickToPlayPatch.cs`

**策略**：Prefix patch `NPlayerHand.StartCardPlay`，在卡牌进入拖拽流程之前，若满足自动出牌条件，直接调用 `TryManualPlay` 出牌。

**实现要点**：
- 类级别 `[HarmonyPatch(typeof(NPlayerHand), "StartCardPlay")]` + 方法级别 `[HarmonyPrefix]`
- 第1次调用（按下）：满足条件 → `TryManualPlay(target)`，设 `_skipNext=true`，返回 `false` 跳过原方法
- 第2次调用（松开）：检测到 `_skipNext` → 清除标志，返回 `false` 跳过，防止重复出牌
- 不满足条件或牌出不了时：返回 `true` 走原拖拽流程
